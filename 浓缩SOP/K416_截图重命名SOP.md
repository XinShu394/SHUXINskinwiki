# K416 截图重命名 SOP（固定流程）

本文档用于规范 K416 截图的分类与重命名，保证后续每次新增截图都能按同一标准落库。

## 1. 适用范围

- 仅适用于 `K416` 目录下的皮肤子文件夹。
- 目标是为每个条目生成标准四图位：`A/B/C/D`。

## 2. 图位定义（固定）

- `A`：市场缩略图，固定来源为 `Snipaste_*.png`
- `B`：市场查看图（外观详情）
- `C`：靶场室内效果图（室内墙面场景）
- `D`：室外效果图（有明显日光）

## 3. 命名格式

统一格式：

`K4-<编码>-<流水号>_<图位>.png`

示例：

- `K4-UQ0109-001_A.png`
- `K4-UQ0109-001_B.png`
- `K4-UQ0109-001_C.png`
- `K4-UQ0109-001_D.png`

## 4. 目录名解析规则（重点）

### 4.1 常规目录

若目录名是标准编码（如 `UQ0109`、`UG0111`）：

- 编码：目录名本身
- 流水号：`001`

示例：

- 目录 `UG0111` -> `K4-UG0111-001_*`

### 4.2 带附加流水号目录

若目录名是“编码 + 三位流水号”（如 `UG0111002`）：

- 编码：前 6 位（`UG0111`）
- 流水号：后 3 位（`002`）

示例：

- 目录 `UG0111002` -> `K4-UG0111-002_*`

错误与正确对照（必须按正确写法）：

- 错误：`K4-UG0111002-002_A.png`
- 正确：`K4-UG0111-002_A.png`
- 规则说明：目录后 3 位是流水号，不属于编码本体。

## 5. 单目录执行步骤

以任意一个皮肤目录为单位：

1. 找到 `Snipaste_*.png`，复制为 `..._A.png`
2. 找到 `Delta Force Screenshot*.png` 三张图，按拍摄顺序分别作为 `B/C/D`
3. 生成标准文件名（见第 3、4 节）
4. 按第 5.1 节执行裁剪
5. 不删除原始截图，只维护标准命名文件

### 5.1 裁剪规则（已通过）

- `A` 图：不裁剪，保持原始 `Snipaste` 画面。
- `B` 图：四边裁剪，保证枪体上下左右有留白，但去掉更多无效背景。
  - 左裁：`15%`
  - 上裁：`10%`
  - 右裁：`6%`
  - 下裁：`18%`
- `C` 图：仅做纵向上裁，不做横向裁剪，保留左上角 `K416 命运` 作为锚点。
  - 上裁：`2.2%`
- `D` 图：仅做纵向上裁，不做横向裁剪，规则与 `C` 一致。
  - 上裁：`2.2%`

备注：

- `C/D` 不要求去掉左上角 `K416 命运`。
- `C/D` 只去掉顶部 UID、温度/丢包等最上方信息。
- 过程中产生的测试文件（如 `_crop`、`_v2`、`_nocrop`）在确认方案后应删除。

## 6. 质检标准

每个目录必须满足：

- 有且仅有 1 张标准 `A` 图
- 有 `B/C/D` 三张标准图
- 文件名符合 `K4-<编码>-<流水号>_<图位>.png`
- 同一目录内四图编码与流水号一致

## 7. 索引文件规范

每次批处理后更新 `K416/k416_index_step1.csv`，至少包含字段：

- `id`（如 `K4-UG0111-002`）
- `folderCode`（原目录名）
- `normalizedCode`（规范编码）
- `serial`（三位流水号）
- `imageA/imageB/imageC/imageD`
- `status`（`ready` 或 `incomplete`）

## 8. 异常处理

- 缺少 `Snipaste`：该目录标记 `incomplete`，人工补图后再重跑
- Delta 图不足 3 张：标记 `incomplete`
- B/C/D 难以自动判断：由人工确认后再命名
- 新增特殊目录命名模式：先补充本文档，再执行批量重命名

## 9. 变更约定

- 任何命名规则变更，先改此 SOP，再执行新一轮批处理。
- 本 SOP 优先级高于旧计划书中的历史命名示例。

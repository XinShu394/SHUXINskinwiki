# 枪皮命名规范与文件夹整理指南

本指南用于让不熟悉项目的人也可以直接完成命名与录入。

目标：
- 统一每个皮肤目录命名
- 一眼看出编码、材质、品级、枪皮名
- 脚本可直接识别并写入网页前端

## 1. 统一目录命名格式

从现在开始，推荐目录名使用：

`<代码目录>__<材质中文>-<品级中文>-<枪皮名>`

示例：
- `UQ0409__其他-优品-海盐橙`
- `UG0111002__贵金属-优品-大渐变`
- `UL七彩雷__镭射-优品-七彩雷`
- `J纯金__NA-极品-纯金`

说明：
- `__` 前面是脚本识别用“代码目录”（核心主键来源，不可乱改格式）
- `__` 后面是给人看的中文说明（材质、品级、枪皮名）
- 脚本会优先使用 `__` 后的“枪皮名”写入前端展示名（若 `meta.js` 已有名称，以 `meta.js` 为准）

## 2. 编码对照表（必须掌握）

### 2.1 品级码
- `U` = 优品
- `J` = 极品

### 2.2 材质码
- `T` = 透光
- `G` = 贵金属
- `Q` = 其他
- `L` = 镭射
- `M` = 漆面
- `Z` = 木质

可复合最多两位（例如 `LZ` = 镭射 + 木质）。

### 2.3 颜色码
- `01` 白
- `02` 红
- `03` 黄
- `04` 青
- `05` 紫
- `06` 棕
- `07` 黑
- `08` 灰
- `09` 橙
- `10` 绿
- `11` 蓝
- `12` 粉
- `00` 单色第二码
- `1111` 未知配色

## 3. 各武器目录“代码目录”规则

### 3.1 K416（编码模式）
- 代码目录：`[U/J][材质码][四位颜色码][可选三位流水]`
- 例：
  - `UQ0409`（默认流水 001）
  - `UG0111002`（编码 UG0111，流水 002）

主键：`K4-<编码>-<流水>`

### 3.2 QBZ95（编码模式）
- 推荐直接使用编码目录（避免中文旧目录歧义）：
  - `JQ0911`
  - `JQ0911002`
- 也兼容历史中文目录，脚本会转换。

主键：`QBZ95-<编码>-<流水>`

### 3.3 腾龙（编码模式 + 特例）
- 代码目录同编码模式：
  - `UG0809`
  - `UQ1101002`
- 历史中文目录兼容，但建议逐步迁移为编码目录。
- 特例 `镭射贵金属` 会映射为 `LG1111`。

主键：`腾龙-<编码>-<流水>`

### 3.4 AUG / KC17 / M4A1（模板+材质模式）
- 代码目录：`[U/J][1-2位材质码][模板名][可选三位流水]`
- 例：
  - `UL七彩雷`
  - `UQ曼巴`
  - `UL模板002`

主键：`武器-<模板名>-<流水>`

### 3.5 SCARH / Vector（纯模板模式）
- 代码目录：`J模板名`（历史默认极品）
- 例：
  - `J纯金`
  - `J薄荷蓝`

主键：`武器-<模板名>-001`

### 3.6 ASVAL（编码模式 + 复合材质）

ASVAL 与 K416/腾龙 编码逻辑一致，但支持 **复合材质**（最多两位材质码，如 `GL/ML/ZL`），且同一编码组内用 **两阶段自动串号**（按文件夹名升序，依次 001/002/003…）。

**文件夹名规则**

| 格式 | 含义 | 示例 |
|---|---|---|
| `[U/J][材质码]` | 单材质，无颜色 | `UG`（贵金属，无特定色） |
| `[U/J][材质码][材质码]` | 复合材质，无颜色（颜色码写 `1111`） | `UGL`（贵金属+镭射） |
| `[U/J][材质码][中文颜色]` | 单材质+颜色 | `UG白`、`UM灰蓝` |
| `[U/J][材质码][中文颜色][数字后缀]` | 同编码第 N 个皮肤 | `UG白002` |
| `[U/J][材质码][材质码][数字后缀]` | 同复合材质第 N 个皮肤 | `UGL001`、`UML002` |

**串号规则**：同 normalizedCode 的目录按名称升序排列，自动分配 001/002/003……  
`UGL`→001，`UGL001`→002，`UGL002`→003（数字后缀只是区分符，不代表最终流水号）

**主键格式**：`ASVAL-<品级><材质码><四位颜色码>-<三位流水>`

**新增 ASVAL 皮肤流程**（两步走）

```
# 步骤 1：图片录入（生成 A/B/C/D 裁剪图）
python scripts/ingest_asval.py

# 步骤 2：校验 + 写入站点
python scripts/validate_and_build.py --weapon ASVAL
```

或在网站首页选择 ASVAL 点"校验并更新"（前提：已手动跑过 ingest_asval.py 生成图片）。

## 4. 图片命名与目录内文件要求

每个皮肤目录必须有标准四图：
- `<id>_A.png`
- `<id>_B.png`
- `<id>_C.png`
- `<id>_D.png`

缺任意一张，脚本会校验失败并中止更新。

## 5. 推荐新建皮肤流程

1. 先按武器规则创建“代码目录”
2. 按规范放置并生成 `A/B/C/D` 标准图
3. 目录名补全为：`代码目录__材质-品级-枪皮名`
4. 运行脚本：
   - 单武器：`python scripts/validate_and_build.py --weapon K416 --normalize-folders`
   - 全量：`python scripts/validate_and_build.py --all --normalize-folders`
5. 检查 `site/reports/validate_warnings.txt` 是否有冲突警告

## 6. 重要边界

- 改名优先通过目录后缀（`__...`）和 `meta.js`，不要改主键 `id`
- 若同一武器出现同 ID 多目录，脚本会优先保留标准编码目录，并在报告中告警
- ASVAL 复合材质（GL/ML/ZL）同组皮肤数量变化时，**串号会整体重排**，请勿手动修改流水号
- ASVAL 新皮肤必须先跑 `ingest_asval.py` 生成 A/B/C/D 图，再跑 validate；其他武器直接 validate 即可

# 三角洲砖皮图鉴 · 部署架构说明

## 整体架构

```
用户浏览器
    │
    ├─ 访问网页/JS/CSS ──→ 阿里云 ECS（Nginx）
    │                        skin.textagent.cn → 8.163.38.4
    │
    ├─ 加载武器图片 ──────→ 阿里云 OSS（传输加速）
    │                        skinwiki.oss-accelerate.aliyuncs.com
    │
    └─ 发布/读取评论 ─────→ ECS 上的 Flask API（5200端口）
                             数据存储在 ECS 本地 SQLite
```

## 三、阿里云 ECS

**公网 IP：** 8.163.38.4

**系统：** Ubuntu 22.04.5 LTS

**代码路径：** `/var/www/zpbk`（从 GitHub clone）

### 运行的服务

| 服务 | 端口 | 说明 |
|---|---|---|
| Nginx | 80 | 反向代理，提供静态文件服务 |
| Flask 评论 API | 5200（仅本地） | 处理评论读写请求 |

### Nginx 配置

- 配置文件：`/etc/nginx/sites-available/zpbk`
- 网站根目录：`/var/www/zpbk/site`
- `/api/` 路径反向代理到 Flask

### 评论数据库

- 类型：SQLite
- 路径：`/var/www/zpbk/server/comments.db`
- 数据完全存储在 ECS 本地，不依赖第三方

## 五、本地开发流程

```bash
# 启动本地开发服务器（双击或命令行）
启动.bat
# 访问：http://localhost:8765/site/index.html

# 同时启动评论 API（如需测试评论功能）
cd server
python app.py
# 评论 API：http://localhost:5200
```

本地开发时图片仍从 OSS 加载（需要网络），评论数据写入本地 `server/comments.db`。

## 六、新增武器皮肤的流程

1. 在本地武器文件夹（如 `K416/`）按规范添加截图
2. 运行构建脚本更新数据：
   ```bash
   python scripts/validate_and_build.py --weapon K416
   ```
3. 将新图片上传到 OSS 对应文件夹
4. 将更新后的代码文件提交到 GitHub：
   ```bash
   git add site/data.js site/covers.js site/meta.js
   git commit -m "新增 K416 xx皮肤"
   git push
   ```
5. 在 ECS 上拉取最新代码：
   ```bash
   cd /var/www/zpbk && git pull
   ```

## 七、ECS 维护常用命令

```bash
# 查看 Nginx 状态
systemctl status nginx

# 重启 Nginx
systemctl reload nginx

# 查看评论 API 是否在运行
ps aux | grep app.py

# 重启评论 API（后台运行）
cd /var/www/zpbk/server
source venv/bin/activate
nohup python app.py > app.log 2>&1 &

# 从 GitHub 拉取最新代码
cd /var/www/zpbk && git pull

# 配置评论 API 开机自启
sudo systemctl enable zpbk-api
sudo systemctl start zpbk-api
```
